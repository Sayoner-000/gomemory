### 2026-06-22 — Mantenimiento de memoria: purgar, compactar, GC, desinstalar + v1.4.0

- **Feature**: `mem purge [flags]` — vacía memorias con alcance acotado al proyecto actual por defecto (`--all` para todos), filtros `--type`/`--older-than-days`, confirmación explícita obligatoria salvo `--yes`
- **Feature**: `mem compact` — `VACUUM` sobre `.memory/mem.db`, reporta tamaño antes/después; nunca borra datos
- **Feature**: `mem gc [flags]` — garbage collection a demanda (retención default 90 días), reutiliza la misma lógica de `purge` en vez de duplicar SQL
- **Feature**: `mem uninstall [dir] [--yes]` — reverso completo de `mem install`: binario, hooks, entradas MCP (`.mcp.json`, `.opencode.json`, `.cursor/`, `.windsurf/`, `.cline/`), bloque de protocolo en AGENTS.md/CLAUDE.md y `.memory/` (datos incluidos); deja `~/.codex/config.toml` intacto (archivo global compartido) e informa al usuario
- **Diseño**: nuevo puerto `application/ports/maintenance_repository.go` separado de `MemoryRepository` (segregación de interfaces — operaciones destructivas no deben mezclarse con el CRUD normal usado por MCP/TUI); explícitamente **fuera de MCP** por ser operaciones destructivas que requieren confirmación humana
- **TUI**: nueva pantalla de mantenimiento (tecla `m`) con Purgar/Compactar/GC; header muestra tamaño en disco junto al conteo de memorias
- **Bug encontrado en validación manual**: `mem uninstall` dejaba un `AGENTS.md`/`CLAUDE.md` con solo el título auto-generado (`# Instrucciones para agentes AI`) cuando el archivo era generado enteramente por `mem install` — corregido con `isAutoGeneratedTitle()` en `cmd_uninstall.go`
- Spec/plan/tasks completos en `specs/003-memory-maintenance/`
- Versión bump: `1.3.0` → `1.4.0` (README, INSTALLATION, plugin.json, MCP server Implementation)
